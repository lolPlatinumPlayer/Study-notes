· state与getter的设计意义
state代表完整的数据，而getter则是从完整数据中提取出需要操作的数据
若没有getter，在反复操作深层级的数据时将要多写很多代码
而且getter可以将需要的数据组合在一起，因此写state时只需要考虑查看整体数据时是否直观，不用担心数据结构在操作时会不够直观
如果在视图中使用state的话，在setter改变后不会触发视图更新（不过用cdn写的进度条测试里会），而用getter的话会


· getter
1、实例的computed里写：
   实例中数据名(){
       return this.$store.getters.store的getters中的函数名;
   }
2、store的getters中写：
   函数名(state,getters){
   /*
   上一行的state和getters分别是是store中的state和getters子项
   因此都是用点（.）语法操作
   */
       return state.state的子项;
       /* 或者
       return getters.其他getter的名称; */
   }


· mapGetters
功能：简化computed中getter代码
使用前须在.vue文件内import { mapGetters } from 'vuex'
例子：
computed: mapGetters({
    想在该.vue文件中代表这数据的名称: 'getter名'
})


· ...mapGetters
功能：可以让computed中插入getter以外内容
...mapGetters可以作为computed的一个属性
使用前提：         （由于...非ES6语法，所以要装插件）
1、cnpm install --save-dev babel-plugin-transform-object-rest-spread
2、.babelrc中加"plugins": ["transform-object-rest-spread"]
复制node_modules须知：
node_modules\_babel-plugin-transform-object-rest-spread@版本号等 及node_modules\_babel-runtime@版本号等 下的node_modules文件夹中的文件夹不知为何会显示快捷方式图标（实际上就是正常文件夹），如果复制其祖先代文件夹，则其中内容不会被复制。
只有选中 显示快捷方式图标的文件夹 进行复制才能复制成功，因此整块node_modules复制完后还需要单独复制这些文件夹。（至少要复制 _babel-plugin-tra.. 中的，不然无法使用）
不过似乎更好的方法是直接cnpm install，更快速，解决问题更多，可以自动补齐残缺文件


· mutation
mutation 都是同步事务（或者说阻塞函数？）
1、实例的methods里写：
   实例中的函数名(传参){ //传参可以是对象及对象字面量
       this.$store.commit('store的mutations的函数名',传参);//【1】
   }
2、store的mutations中写：
   函数名(s,传参){//【2】
       //操作state中数据时写：s.要操作的数据名
       //操作getter中数据时写：this.getters.要操作的数据名
       //操作state和操作getter结果上看似乎没区别
       //这里的传参与实例的methods中的传参对应
   }
去掉传参也可以
【1】【2】处的传参也叫荷载（payload），且数量不能超过一个
因此要传多个数据应该使用对象或者数组


...mapMutations([
    'a',  //要使用的method与Mutation同名
    b:'a' //将a Mutation放入b method中
          //可以提交荷载，甚至用字面量都可以提交
]),


· Action
提交（在实例中）：
methods: {
    使用时想用的函数名() {
        this.$store.dispatch('存放于actions中的action名')
    }
}
注册：
actions:{
    action名(context) {
        //setTimeout(() => {
            context.commit('mutation名')
        //}, 600)
    }
} //这个例子效果和直接使用mutation完全一致，就算加上setTimeout也是完全一致（mutation也能用setTimeout）
但是如果在mutation中使用异步脚本，vue-devtools无法在“时光旅行”中捕捉到那次mutation延时改变的数据，而action是异步结束后才生成“时光旅行”记录，所以能捕捉到异步后的数据。这就是【action存在的意义】
辅助函数和mutation等一致
